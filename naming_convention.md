# Naming convention

- Camel case
- Acronyms are considered as single words.
  - e.g. `Llvm`, `Ast`, etc.

## Types

- `Null`
- `Number`
- `Foo`
- `Bar`

## Variables

- `foo`
- `fooBar`

## Packages

The last components should be in kebab case (?).

- `github.com/ein-lang/foo`
- `github.com/foo/bar`
