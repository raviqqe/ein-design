# v10

- Nominal typing
- No top type

## Built-in types

### Primitives

```
Boolean
None
Number
String
```

### Arrays

```
[a]
```

#### Update

```
[...a, 42]
[42, ...a]
```

### Maps

```
{a}
```

#### Update

```
{ ...a, foo = "bar" }
```

### Functions

```
a -> b
```

## User-defined types

### Record types

- Encapsulated
  - Properties are hidden by default.

```
type Person =
  { name : String
  , age : Number
  }
```

#### Exporting properties

```
export { Person { ... } }
```

#### Initialization

```
Person { name = "foo", age = 42 }
```

#### Property access

```
.name person
```

#### Update

```
{ ...person, name = "bar" }
```

### Union types

- Type aliases
- No statically typed enumeration
  - Like Go
  - Can it be done by `type Foo = {}`?

```
type Foo =
    Bar
  | Baz
  | Blah
```

#### Extension

```
type Foo =
    Bar
  | Baz

type Blah =
    Foo
  | Qux
```

## Special types

### Result types

```
type Result =
    Foo
  | Error
```

#### Error types

- All types suffixed with `Error`
  - e.g. `ParseError`

#### ! unary operator

- Return all error types in union types immediately.

```
bar : String -> String | Error
baz : String -> String -> String
error : String -> Error

foo : String -> String | Error
foo x =
  baz (! bar x) (! error "fail always")
```

### Option types

```
type Option =
    Foo
  | None
```

## Case expression

```
case foo of
  { name = "John" } -> e1
  { name = "Doe" } -> e2
  { name } -> e3
```

### Type switch

```
case bar of
  Foo -> e1
  Bar -> e2
  Baz -> e3
```
