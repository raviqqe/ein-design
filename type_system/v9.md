# v9

- Nominal typing

## Built-in types

### Values

```
Boolean
None
Number
String
```

### Arrays

```
[a]
```

### Maps

```
{ a }
```

### Functions

```
a -> b
```

### Any

- The top type

```
Any
```

## User-defined types

### Record types

- Encapsulated
  - Properties are hidden by default.

```
type Person =
  { name : String
  , age : Number
  }
```

#### Exporting properties

```
export { Person { ... } }
```

### Union types

- Type aliases
- No statically typed enumeration
  - Like Go
  - Can it be done by `type Foo = {}`?

```
type Foo =
    Bar
  | Baz
  | Blah
```

#### Extension

```
type Foo =
    Bar
  | Baz

type Blah =
    Foo
  | Qux
```

## Result types

```
type Error =
  { error : None // Error indicator field
  , context : Any
  }

type Result =
    Foo
  | Error
```

### ! unary operator

```
bar : String -> String | Error
baz : String -> String -> String
error : String -> Error

foo : String -> String | Error
foo x =
  baz (! bar x) (! error "fail always")
```

## Option types

```
type Option =
    Foo
  | None
```

## Case expression

```
case foo of
  { name = "John" } -> e1
  { name = "Doe" } -> e2
  { name } -> e3
```

### Type switch

```
case bar of
  Foo -> e1
  Bar -> e2
  Baz -> e3
```
